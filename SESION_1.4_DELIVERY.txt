================================================================================
                           API ENDPOINTS (NUEVOS)
================================================================================

1. GET /api/recomendaciones/{operario_id}

   Retorna: Recomendaciones automáticas basadas en los 5 análisis

   Response:
   {
     "operario_id": "OP_00045",
     "total_recomendaciones": 3,
     "recomendaciones": [
       {
         "tipo": "caida_progresiva",
         "titulo": "🔴 ALERTA: Fatiga Detectada",
         "severidad": "CRÍTICA",
         "descripcion": "Operario pierde 22% de velocidad durante turno",
         "recomendacion": "Ofrecer pausa preventiva en próxima hora",
         "impacto": "Prevenir pérdida de ~100 picks",
         "accionable": true,
         "accion": "trigger_pausa"
       },
       ...
     ],
     "resumen": {
       "total_picks": 5200,
       "velocidad_promedio": 24.3,
       "tasa_error_pct": 2.1
     }
   }

2. GET /api/alertas?dias=1

   Retorna: Alertas activas de los últimos N días

   Response:
   {
     "total_alertas": 2,
     "periodo_dias": 1,
     "alertas": [
       {
         "timestamp": "2026-04-20T16:45:30.123456",
         "operario_id": "OP_00045",
         "tipo": "tasa_error_alta",
         "severidad": "CRÍTICA",
         "mensaje": "Tasa error 6.2% (esperado: 2-3%)",
         "accion_sugerida": "Revisar con operario"
       }
     ]
   }

3. POST /api/config/guardar

   Guarda configuración de umbrales y alertas

   Body:
   {
     "caida_critica": 20,
     "caida_alta": 15,
     "error_maximo": 5,
     "velocidad_minima": 28,
     "alertas_activas": ["caida", "error", "especialidad"],
     "email_destino": "supervisor@coto.com.ar",
     "frecuencia_reporte": "diaria"
   }

   Response:
   {
     "status": "success",
     "mensaje": "Configuración guardada",
     "config_guardada": {...}
   }

4. GET /api/reportes/generar?formato=json&tipo=diario&incluir=all

   Genera reportes en diferentes formatos

   Query Parameters:
   - formato: json, pdf, excel
   - tipo: diario, semanal, mensual
   - incluir: ranking, graficos, anomalias, all

   Response (JSON):
   {
     "fecha_generacion": "2026-04-20T16:45:30.123456",
     "tipo": "diario",
     "periodo": "últimas 24 horas",
     "resumen": {
       "total_operarios": 10,
       "total_picks": 52000,
       "total_bultos": 125000,
       "promedio_velocidad": 24.5
     },
     "ranking": [
       {
         "rank": 1,
         "operario_id": "OP_00045",
         "picks": 5200,
         "bultos": 12500,
         "velocidad": 24.3,
         "errores": 2
       }
     ]
   }

================================================================================
                           CÓMO USAR (USUARIO)
================================================================================

1. ABRIR DASHBOARD
   http://localhost:8080/config_y_recomendaciones
   o
   http://localhost:8080/config_y_recomendaciones.html

2. CONFIGURAR UMBRALES (Tab 1)
   - Mover sliders para caida_critica, caida_alta, error_maximo, etc.
   - Los valores se actualizan en tiempo real
   - Ver valores actuales bajo cada slider
   - Clickear "Guardar Configuración" para persistir
   - Botón "Restaurar Defaults" para volver a valores originales

3. VER RECOMENDACIONES (Tab 2)
   - Selector dropdown para elegir operario
   - Dashboard carga automáticamente recomendaciones
   - Cards muestran cada recomendación con:
     * Ícono + título (ej: "🔴 ALERTA: Fatiga Detectada")
     * Descripción: qué está pasando
     * Sugerencia: acción a tomar
     * Impacto estimado: beneficio de la acción
     * Severity badge: CRÍTICA (rojo), ALTA (naranja), POSITIVO (verde), etc.
   - Botón de acción clickeable para cada recomendación

4. MONITOREAR ALERTAS (Tab 3)
   - Auto-refresh cada 10 segundos
   - Muestra todas las alertas activas del último período
   - Ordenadas por severidad
   - Cada alerta muestra: operario, tipo, mensaje, acción sugerida
   - Indicador visual de severidad

5. GENERAR REPORTES (Tab 4)
   - Selector de tipo: diario, semanal, mensual
   - Checkboxes para seleccionar formatos (JSON, PDF, Excel)
   - Botón "Generar Reporte"
   - Sistema genera y descarga archivo
   - Historial de reportes generados

================================================================================
                    ALGORITMO DE RECOMENDACIONES
================================================================================

Función: get_recomendaciones_operario(operario_id)

1. Obtener datos operario:
   - total_picks, velocidad_promedio, tasa_error

2. Ejecutar 5 análisis inteligentes:
   - caida_progresiva: ¿Está fatigándose?
   - correlacion_sku: ¿Tiene SKUs favoritos?
   - patron_semanal: ¿Hay patrones por día?
   - recuperacion_pausa: ¿Las pausas ayudan?
   - anomalia_zscore: ¿Hay picks raros?

3. Para cada análisis, generar recomendación:
   - Si caída > 20%: CRÍTICA "Fatiga Detectada" → "Ofrecer pausa"
   - Si caída > 15%: ALTA "Cansancio Moderado" → "Monitorear"
   - Si especialidad > 30%: POSITIVO "Especialización" → "Aumentar SKU"
   - Si variación semanal > 15%: INFO "Patrón Semanal" → "Rebalancear"
   - Si recuperación > 20%: POSITIVO "Pausas Estratégicas" → "Aumentar pausas"
   - Si anomalías > 5%: ALERTA "Picks Anormales" → "Investigar"
   - Si tasa_error > 5%: CRÍTICA "Tasa Error Alta" → "Revisar capacitación"

4. Retornar lista ordenada por severidad
